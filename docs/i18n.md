# Hệ thống i18n hiện tại

## Tổng quan

Repo hiện dùng mô hình `locale-aware routing` với toàn bộ app thật nằm dưới `src/app/[locale]`.

Các route chính:

- `/{locale}`
- `/{locale}/login`
- `/{locale}/admin`
- `/{locale}/admin/blog-departments`
- `/{locale}/admin/settings`
- `/{locale}/dev/doc`

Trong đó:

- `locale` hiện hỗ trợ: `vi`, `en`
- URL là nguồn locale chuẩn
- Bare URL như `/`, `/login`, `/admin`, `/dev/doc` không còn có page riêng trong App Router
- `proxy.ts` là nơi duy nhất redirect bare URL sang route có locale

## Kiến trúc hiện tại

```text
src/
├── app/
│   ├── layout.tsx                 # root shell tối thiểu
│   └── [locale]/

│       ├── login/page.tsx

```

## Luồng locale

### 1. Request đi vào `proxy.ts`

`src/proxy.ts` xử lý các việc sau:

- Nếu request là `/`
  - lấy locale theo thứ tự:
    1. `User.language` từ Frappe nếu đang có session
    2. cookie `cds_app_locale`
    3. fallback `vi`
  - redirect sang `/{locale}`
- Nếu request chưa có prefix locale
  - redirect sang đường dẫn có locale bằng `buildLocalePath()`
- Nếu request đã có locale hợp lệ
  - tiếp tục request
  - set lại cookie `cds_app_locale`
  - inject header `x-locale` cho server render
- Đồng thời vẫn áp dụng auth guard:
  - route public: `/login`, `/register`
  - route private: các path còn lại sau khi đã strip locale

### 2. Root layout

`src/app/layout.tsx` không còn bootstrap i18n nữa.

Nó chỉ còn:

- import font
- import `globals.css`
- set `<html lang>` theo header `x-locale`
- render `{children}`

Lý do root layout vẫn tồn tại:

- đây là yêu cầu bắt buộc của App Router
- `html lang` chỉ có thể được set ở root layout

### 3. Locale layout

`src/app/[locale]/layout.tsx` là i18n boundary thật của app.

Nó chịu trách nhiệm:

- validate `params.locale`
- lấy dictionary bằng `getDictionary(locale)`
- mount `ApiProvider locale={locale} dictionary={dictionary}`
- cấu hình metadata theo locale

Nói ngắn gọn:

- root layout: shell tối thiểu
- `[locale]/layout.tsx`: toàn bộ bootstrap app thật

## Client-side language context

### `src/contexts/LanguageContext.tsx`

Context hiện chỉ tiêu thụ dữ liệu đã được server cấp xuống.

State chính:

- `locale`
- `language`
- `t`
- `supportedLanguages`

Hàm `setLanguage(lang)` hiện:

- validate locale
- build URL mới bằng `buildLocalePath()`
- cập nhật cookie `cds_app_locale`
- `router.push()` sang locale mới

Hệ thống hiện không dùng `localStorage` để lưu locale.

### `src/hooks/useLanguage.ts`

Hook này là wrapper mỏng quanh `LanguageContext`, trả ra:

- `locale`
- `language`
- `setLanguage()`
- `t`
- `supportedLanguages`
- `isLanguage()`

## Cách dùng trong component

### Đọc text theo locale

```tsx
'use client';

import { useLanguage } from '@/hooks/useLanguage';

export function Example() {
  const { t } = useLanguage();

  return <h1>{t.admin.dashboardTitle}</h1>;
}
```

### Tạo link giữ nguyên locale hiện tại

```tsx
'use client';

import Link from 'next/link';
import { useLanguage } from '@/hooks/useLanguage';
import { buildLocalePath } from '@/i18n';

export function ExampleLink() {
  const { locale } = useLanguage();

  return (
    <Link href={buildLocalePath(locale, '/admin')}>
      Admin
    </Link>
  );
}
```

### Đổi ngôn ngữ

```tsx
'use client';

import { useLanguage } from '@/hooks/useLanguage';

export function LocaleSwitcher() {
  const { setLanguage } = useLanguage();

  return (
    <>
      <button onClick={() => setLanguage('vi')}>VI</button>
      <button onClick={() => setLanguage('en')}>EN</button>
    </>
  );
}
```

## Những nguyên tắc hiện tại

- App thật chỉ tồn tại trong `src/app/[locale]`
- Không giữ page redirect ở `src/app/page.tsx`, `src/app/login/page.tsx`, `src/app/admin/*`, `src/app/dev/doc/page.tsx`
- Bare URL chỉ được xử lý bởi `proxy.ts`
- URL có locale luôn được ưu tiên hơn cookie
- `User.language` chỉ dùng để bootstrap redirect từ `/`
- `setLanguage()` đổi locale bằng navigation sang URL mới
- Tất cả internal link nên dùng `buildLocalePath(locale, path)`
- Không dùng `localStorage` làm nguồn locale

## Khi thêm màn hình mới

Checklist:

1. Tạo page dưới `src/app/[locale]/...`
2. Dùng `useLanguage()` nếu là client component cần text UI
3. Dùng `buildLocalePath(locale, path)` cho mọi link nội bộ
4. Nếu page có metadata riêng, thêm `generateMetadata()` theo locale
5. Thêm text mới vào cả `en.ts` và `vi.ts`

## Khi thêm locale mới

Các bước tối thiểu:

1. Tạo dictionary mới, ví dụ `src/i18n/fr.ts`
2. Khai báo locale mới trong `src/i18n/index.ts`
3. Cập nhật `SUPPORTED_LOCALES`
4. Cập nhật `SUPPORTED_LANGUAGES`
5. Cập nhật `generateStaticParams()` trong `src/app/[locale]/layout.tsx`
6. Kiểm tra lại `proxy.ts` và redirect logic

## Ghi chú kỹ thuật

- `html lang` hiện được lấy ở root layout từ header `x-locale` do `proxy.ts` set
- Đây là chỗ duy nhất root layer còn “biết” đến locale
- Provider, dictionary và app bootstrap thật đã nằm trong `[locale]/layout.tsx`
